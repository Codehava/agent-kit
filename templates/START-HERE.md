# START HERE

Panduan ini untuk pengguna non-coder. Jika Anda bingung harus mulai dari mana, ikuti jalur ini saja.

## 1. Jangan Mulai dari Teknologi

Sebelum bicara framework, siapkan jawaban untuk:
- Siapa pengguna utama aplikasi ini
- Masalah apa yang ingin diselesaikan
- Apa hasil utama yang harus user capai
- Tiga fitur wajib untuk MVP
- Deadline atau target uji coba pertama

## 2. Jalur Kerja yang Disarankan

Gunakan hanya 5 command ini:

1. `/vibe-plan`
2. `/launch`
3. `/apply`
4. `/unify`
5. `/progress`

Workflow lain bersifat lanjutan. Abaikan dulu jika Anda baru mulai.

## 3. Prompt Pertama

```text
Saya non-coder.
Saya ingin membangun [jenis aplikasi] untuk membantu [siapa]
agar mereka bisa [hasil utama].
Tolong bantu saya mulai dari /vibe-plan.
Pilih solusi paling sederhana untuk MVP dan tanyakan satu hal setiap langkah.
```

## 4. Kapan Lanjut ke Coding

Baru lanjut ke coding setelah Anda setuju dengan:
- `docs/README.md`
- `PLANNING.md`

Kalau masih ada bagian yang terlalu rumit, potong scope dulu.

## 5. Aturan Aman

- Jangan anggap semua contoh template wajib dipakai
- Hapus bagian payment, mobile, realtime, marketplace, atau compliance khusus jika tidak relevan
- Jika AI memberi solusi yang terasa terlalu rumit, minta versi yang lebih sederhana
- Gunakan `/progress` jika bingung harus lanjut apa

## 6. Kalau Bingung Harus Mengetik Apa

Buka `KEYWORDS-CHEATSHEET.md`.
Di situ ada command dan prompt yang paling aman dipakai.

## 7. Kalau Ingin Lihat Urutan Dokumen

Buka `docs/README.md`.
Di situ ada peta dokumen bisnis dan dokumen build inti.

Kalau hanya ingat satu hal: `rencana yang sederhana lebih berharga daripada stack yang terlihat canggih`.
